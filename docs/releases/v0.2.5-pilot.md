1. 현재 downstream의 `v0.2.4-pilot` tag·commit·archive checksum과 project override를 기록한다.
2. 게시된 `v0.2.5-pilot` archive checksum을 release note와 대조한다.
3. 격리 target에서 기존 canonical lock과 새 release manifest를 사용해 upgrade preview를 실행한다.
4. capability·quality·adapter profile의 project별 tool version·network·비용·reviewer 값을 명시한다.
5. full-stack materializer는 frontend·backend·shared target과 migration up/down artifact diff를 검토한다.
6. 명시 승인 후 filesystem materialization만 적용하고 공통·project regression을 통과한다.
7. 검증 중 rollback record를 유지하고 승인 완료 뒤 finalize한다.

이 release는 dependency install, live model 호출, 실제 FastAPI runtime 호출, DB migration 실행,
Production write를 자동 승인하지 않는다.
