# v0.2.2-pilot

상태: GitHub prerelease 발행 완료
발행일: 2026-07-11

## 목적

`v0.2.1-pilot` 이후 downstream에서 발견한 validator lint 문제와 설치된 package version 변경의
Human-in-the-loop 계약을 release baseline에 포함한다.

## 변경

- downstream validator의 미사용 `execFileSync` import 제거
- REQ-034: 설치된 package version 변경 전 사용자 승인 필수
- dependency upgrade workflow 추가
- update·downgrade·override·transitive·lockfile-only 변경을 모두 승인 대상으로 정의
- current·target version, migration·호환성·코드 영향·lockfile diff·검증·rollback preview 요구
- 적용 중 예상하지 못한 dependency·lifecycle·telemetry·network·권한 변화 시 재승인 요구
- token-aware와 full에서 같은 승인·호환성·회귀 gate 유지

## Migration

- `v0.2.1-pilot`의 실행 정책과 manifest schema는 호환된다.
- downstream의 `scripts/validate-downstream.mjs` local lint override는 이 release에서 제거할 수 있다.
- 기존 package version은 자동 변경하지 않는다. 이 release 적용 자체가 application dependency
  update를 승인하는 것으로 해석되지 않는다.

## 검증

- bootstrap·downstream validator positive/negative fixture
- HANDOFF staged·range fixture
- `.env*`와 미승인 MCP 차단 회귀
- npm audit, Gitleaks와 Opengrep full scan
- 원격 clean clone bootstrap·validate

## Rollback

Downstream은 현재 upstream lock을 보존하고 별도 upgrade commit에서 적용한다. 실패하면 파일별
삭제 대신 upgrade commit을 revert하고 이전 tag·checksum으로 돌아간다.
