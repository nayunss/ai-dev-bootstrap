# v0.2.0-pilot

상태: GitHub prerelease 발행 완료
발행일: 2026-07-11

## 목적

첫 downstream Todo pilot에서 확인한 보안·호환성·설치 문제를 공통 upstream 정책과 실행 가능한
bootstrap·validate gate로 전환한다. 이 release는 자동 upgrade가 아니며 downstream adopter가
diff와 권한을 검토하고 승인한 경우에만 적용한다.

## 주요 변경

- AI의 `.env*` 읽기·검색·색인·MCP 접근 default-deny
- 미심사 MCP server·tool·설정 변경 default-deny와 승인 manifest
- project-local `.editorconfig`와 코드 스타일 일관성 기준
- task commit·PR의 `HANDOFF.md` 갱신 자동 gate
- 변경 없는 `scripts/bootstrap preview`
- scripts-off·strict-peer dependency bootstrap과 Corepack fallback 금지
- exact engine·package manager·dependency, lockfile, pnpm 11 override, Next telemetry 검증
- network 분리형 dependency vulnerability audit
- Playwright browser와 Husky의 별도 preview·Human-in-the-loop 유지
- clean-clone·positive/negative fixture 회귀 테스트

## 호환성과 Migration

- 기존 `v0.1.0-pilot` downstream은 자동으로 변경되지 않는다.
- `.ai`, `.claude`, `.codex`, `.gitleaks.toml`, `.editorconfig`, 공통 scripts와 security rule 변경을
  project-local 파일로 preview한 뒤 적용한다.
- downstream 고유 설정과 애플리케이션 코드는 덮어쓰지 않고 충돌을 수동 병합한다.
- `approved-mcp.json`은 빈 allowlist로 시작하므로 기존 MCP는 재심사 전까지 호출이 차단된다.
- `.env.example`도 `.env*` 차단 대상이다. AI에 필요한 변수 이름은 별도 비민감 schema 문서로
  제공한다.
- 새 HANDOFF gate는 task 변경과 같은 commit·PR에 실제 맥락 갱신을 요구한다.

## Rollback

downstream 적용 전 branch 또는 worktree를 만들고 기존 `upstream.lock.yaml`을 보존한다. 검증 실패
시 새 파일과 설정을 임의 삭제하지 말고 적용 commit을 revert하거나 이전 release pin으로 되돌린다.
credential·MCP scope가 변경됐다면 rollback과 별도로 해당 credential을 회수·교체한다.
