## Migration

1. 현재 downstream의 upstream release·commit·archive checksum과 local override를 기록한다.
2. 게시된 `v0.2.8-pilot` archive checksum과 release note evidence를 대조한다.
3. 공통 adoption core에서 profile·skill·adapter 선택을 preview한다.
4. 기존 파일 충돌이나 project별 필수 결정이 남아 있으면 적용하지 않는다.
5. 적용 후 lock·target hash와 공통·project regression을 검증한다.
6. 실패하면 transaction rollback 또는 이전 `v0.2.7-pilot` pin으로 복귀한다.

## Rollback

- 적용 전 preview와 승인 경계를 유지한다.
- installer가 생성했고 hash가 변하지 않은 파일만 자동 원복한다.
- 기존 동일 파일과 사용자가 변경한 파일은 보존한다.
- dependency·DB·provider·Production rollback은 이 filesystem rollback과 별도 승인·증거가 필요하다.
