# v0.2.9-pilot

상태: GitHub prerelease 발행 준비
기준일: 2026-07-19

## 목적

GitHub 저장소를 사용하는 비개발자용 기본 surface를 desktop installer가 아닌 GitHub App Web
Portal로 구체화한다. 실제 App credential이나 provider write 없이 권한·token·webhook·승인·PR
경계를 검증할 수 있는 local reference를 `v0.2.8-pilot` 이후 baseline으로 제공한다.

## 구현 범위

- selected-repository installation과 installation account·사용자 push 권한 재검증
- Metadata read, Contents·Pull Requests write의 exact App permission allowlist
- one-time OAuth state, webhook HMAC-SHA256·delivery replay와 bounded state/session retention
- installation token의 ephemeral checkout callback, user access token의 PR delivery callback 격리
- preview actor·repository·plan SHA-256·TTL binding과 actor-bound idempotency
- default branch push·force push·auto merge 없는 branch·Pull Request delivery 계약
- 반응형 한국어 Portal, keyboard semantic HTML과 loopback no-network demo server
- GitHub Actions P0·공통 release adoption core 회귀와 CI security gate 연결

## 검증 경계

다음 local·deterministic fixture는 PASS했다.

- all-repository·권한 확대·suspended installation·사용자 push 권한 철회 차단
- auth state 재사용·만료·open redirect와 webhook signature·event·duplicate delivery 차단
- token 형식·길이 비가정, callback 밖 노출과 nested log 문자열 redaction
- stale plan·actor/repository 교체·중복 request와 session capacity negative
- preview→exact-plan apply→PR-only 결과, merge `NOT_RUN`
- loopback CSP·no-store·same-origin·body limit, semantic HTML과 local simulation 표시
- Gitleaks·Opengrep, requirement·HANDOFF traceability와 CodeSight regeneration

다음은 이 release로 승인하거나 실행하지 않는다.

- 실제 GitHub App 등록·권한 변경·OAuth callback·webhook endpoint
- GitHub provider API, 실제 repository checkout·branch·Pull Request와 rollback
- hosting·persistent replay store·secret manager·ephemeral compute와 egress enforcement
- 실제 suspension·uninstall·authorization revoke와 incident kill switch
- 모바일·desktop 비개발자 2명, screen reader와 실제 PR review 사용성 Eval
- Production 배포·자동 merge·DB migration·dependency install·telemetry
- desktop signing·notarization·installer publish

## Migration

1. 기존 `v0.2.8-pilot` release·commit·archive checksum과 local override를 기록한다.
2. 게시된 `v0.2.9-pilot` tracked archive checksum을 아래 Artifact evidence와 대조한다.
3. 실제 GitHub 연결 없이 `npm run portal:demo`로 local reference 흐름을 확인한다.
4. App owner·operator·incident owner, installation account, hosting·secret·retention이 정해지지
   않았으면 Production pilot을 시작하지 않는다.
5. 실제 App 등록·권한·callback·webhook·배포는 각각 별도 승인하고 non-production repository에서
   먼저 검증한다.

## Rollback

- Local reference 도입 중 문제가 생기면 해당 적용 commit을 revert하고 `v0.2.8-pilot` pin으로
  돌아간다.
- 이 release는 실제 GitHub App이나 provider resource를 만들지 않으므로 외부 credential·installation
  rollback을 수행하지 않는다.
- 향후 Production pilot rollback은 App suspend·uninstall, token revoke, 생성 branch·PR 정리와
  evidence 보존을 별도 runbook으로 검증한 뒤 실행한다.
- 사용자 repository, 기존 파일, default branch와 사용자 untracked 파일을 자동 삭제하지 않는다.
