## Migration

1. 현재 upstream tag·commit·checksum과 downstream local override를 기록한다.
2. `v0.2.3-pilot` archive를 격리 workspace에 풀고 기존 project와 파일별 diff를 검토한다.
3. `scripts/bootstrap preview <target>`으로 root·하위 manifest, application inventory와 readiness profile
   누락을 확인한다.
4. 다중 application project는 `docs/development-environment.md` JSON inventory와 application별
   quality·CI·deploy·hook evidence를 추가한다.
5. Production readiness가 적용되고 profile이 없는 기존 project만 `scripts/bootstrap readiness <target>`을
   명시적으로 실행한다. 기존 profile은 자동 overwrite·merge하지 않는다.
6. dependency·hook·browser binary·provider 변경은 이 release 적용과 별도로 정확한 version·영향을
   승인받는다.
7. 공통·project 회귀와 `scripts/validate <target>`을 통과한 별도 upgrade commit으로 적용한다.

기존 application dependency, Production infrastructure와 database는 자동 변경하지 않는다. readiness
template은 생성 직후 `TBD` 때문에 Production을 차단하며 사람의 근거 없이 READY로 승격하지 않는다.

## Rollback

- Downstream은 새 파일을 개별 삭제하지 않고 별도 upgrade commit 전체를 revert한다.
- readiness profile에 project 결정을 기록했다면 이전 upstream pin으로 돌아가도 해당 정책 문서는
  보존하고 schema 차이만 수동 검토한다.
- application code rollback과 DB migration/provider restore rollback을 분리한다. 이 release는
  Production DB migration이나 provider restore 권한을 부여하지 않는다.
- 실패하면 이전 `v0.2.2-pilot` tag와 게시 checksum으로 돌아간다.
