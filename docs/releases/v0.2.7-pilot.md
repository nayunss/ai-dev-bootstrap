# v0.2.7-pilot

상태: 발행 준비
발행 예정일: 2026-07-17

## 목적

`env-be`의 UF-001–UF-013을 project·stack 고유 구현에서 분리해 공통 원인으로 일반화하고, 중복
finding을 단일 primary REQ와 안정적인 구현·재검증 task로 연결한다. 어떤 AI 도구를 사용해도 같은
공통 source·schema·validator·fixture로 동일한 판정과 필수 산출물을 얻는 tool-neutral 후속 구현
우선순위를 고정한다.

이 release는 요구사항·작업 추적 설계 baseline이다. REQ-049–REQ-052 validator·schema·fixture,
requirements–HANDOFF 교차 검증, 전체 REQ manifest, feedback triage automation과 실제 downstream
재검증의 구현 완료를 의미하지 않는다.

## 승인된 변경

- UF-001–UF-006·UF-011을 REQ-049 정책 주장–구현–반증 가능한 evidence 연결로 통합
- UF-007–UF-008을 REQ-050 staged tree·check-only repository invariant로 통합
- UF-009–UF-010을 REQ-051 push·CI·deploy·health·behavior·Production 상태 분리로 통합
- UF-012를 REQ-052 cross-stack 사용자 표시 언어·locale 계약으로 일반화
- UF-013을 REQ-046 upstream release·commit·archive checksum baseline 계약으로 보강
- 공통 구현과 실제 downstream 재검증을 별도 task로 유지하고 synthetic PASS의 해결 승격을 차단
- requirements–HANDOFF reconciliation, workflow rule, 전체 추적성, tool-neutral triage automation을
  최우선 공통 구현 순서로 확정
- Markdown REQ 범위를 렌더링 안전한 en dash 표기로 정리

## 구현·검증 경계

다음 항목은 설계 승인 또는 task 분류 상태이며 `NOT-RUN`이다.

- REQ-049 policy evidence schema·validator·fixture
- REQ-050 staged tree와 check-only source mutation validator·fixture
- REQ-051 delivery evidence 상태 schema·provider-neutral adapter·fixture
- REQ-052 full-stack locale adapter·cross-stack fixture
- requirements–HANDOFF·feedback task 교차 검증
- REQ-001–REQ-052 전체 machine-readable traceability manifest
- downstream feedback triage workflow·schema·validator와 재사용 skill
- 새 release 기준 실제 `env-be` UF-001–UF-013 항목별 재검증

기존 v0.2.6까지의 reference automation과 hosted gate PASS는 유지하지만 위 신규 구현이나 실제
downstream 해결의 증거로 확대하지 않는다.

## Migration

1. 현재 downstream의 upstream release·commit·archive checksum과 local override를 기록한다.
2. 게시된 `v0.2.7-pilot` archive checksum을 이 문서의 Artifact와 대조한다.
3. requirements, feedback triage와 HANDOFF diff를 검토하되 project 고유 정책·stack을 공통 예시로
   교체하지 않는다.
4. REQ-049–REQ-052 구현 전에는 이 release를 문서·작업 baseline으로만 사용한다.
5. 실제 UF 재검증은 관련 구현을 포함한 후속 release를 고정한 뒤 수행하고 항목별 evidence를 남긴다.

## 검증

- HANDOFF 상단·하단 stable task ID와 우선순위 일치
- REQ-001–REQ-052 문서 상태·추적 링크와 latest requirement metadata
- UF-001–UF-013 단일 primary REQ·구현 task·재검증 task mapping
- Markdown local link와 렌더링 안전한 REQ 범위 표기
- CodeSight regenerate 후 generated diff
- Gitleaks·Opengrep와 hosted `security`·`license-provenance`
- tracked archive checksum과 게시 asset 재다운로드 검증

## Rollback

- 이 release는 downstream source·dependency·DB·provider를 자동 변경하지 않는다.
- 문서 baseline 적용 중 충돌하면 적용 commit을 revert하고 `v0.2.6-pilot` pin으로 돌아간다.
- 기존 downstream finding과 evidence를 삭제하지 않고 새 release 기준 재관찰 결과를 별도 기록한다.
- 사용자 파일, project policy와 도구별 개인 설정을 자동 덮어쓰거나 삭제하지 않는다.
