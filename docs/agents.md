# 에이전트 체계

상태: 설계 승인

## 결론

에이전트는 많이 만드는 것보다 **역할과 권한 경계를 명확히 하는 것**이 중요하다.
기본값은 하나의 구현 에이전트이며, 독립 검증이나 전문성이 실제로 필요한 경우에만 역할을 분리한다.
성격형 persona와 작업 역할의 선택 기준은 [AI 페르소나와 작업 역할 가이드](persona-and-role-guidelines.md)를
따른다.

## 권장 역할

| 역할 | 주요 책임 | 기본 권한 | 대표 산출물 |
|---|---|---|---|
| `coordinator` | 요구사항 확인, 계획, 작업 범위와 완료 조건 관리 | 읽기, 작업 배정 | 계획, 진행 상태 |
| `architect` | 대안 비교, 인터페이스와 ADR 작성 | 읽기, 문서 작성 | 설계, ADR |
| `implementer` | 코드와 테스트 구현 | 작업 브랜치 쓰기 | 코드, 단위 테스트 |
| `reviewer` | 결함, 회귀, 유지보수성 검토 | 기본 읽기 전용 | 리뷰 보고서 |
| `security-reviewer` | 위협과 권한, 비밀정보, 공급망 검토 | 읽기 전용 | 보안 보고서 |
| `release-manager` | 릴리스 준비, 승인 확인, 롤백 점검 | 제한된 배포 권한 | 릴리스 기록 |

작은 작업에서는 `coordinator`, `architect`, `implementer`를 하나의 에이전트가 수행할 수 있다.
고위험 변경에서는 구현과 최종 검증을 분리한다.

## 에이전트 계약

각 역할 문서는 다음을 선언한다.

- 책임과 책임 밖의 일
- 필요한 입력과 접근 가능한 경로
- 실행 가능한 명령과 금지된 작업
- 기대 산출물과 전달 형식
- 완료 조건
- 사용자 승인 또는 다른 역할로 넘겨야 하는 조건

## 권한 모델

| 위험 수준 | 예시 | 정책 |
|---|---|---|
| 낮음 | 파일 검색, 테스트 실행, 문서 초안 | 자동 수행 가능 |
| 중간 | 코드 수정, 의존성 잠금 변경, 마이그레이션 작성 | 범위 내 수행 후 검증 필수 |
| 높음 | 운영 배포, 데이터 삭제, 비밀 접근, 외부 메시지 전송 | 사람의 명시적 승인 필수 |

에이전트는 최소 권한을 사용하며, 권한 부족을 우회하지 않는다. 작업 범위를 벗어난 변경과
기존 사용자 변경을 발견하면 보존하고 충돌 여부를 보고한다.

## 위임 규칙

- 독립적으로 검증 가능한 작업만 위임한다.
- 같은 파일을 동시에 수정하는 역할 배정은 피한다.
- 위임할 때 입력, 범위, 산출물, 완료 조건을 함께 제공한다.
- 최종 책임은 조정 역할에 있으며 결과를 통합하고 다시 검증한다.
- 단순한 작업을 역할 수만 늘려 분할하지 않는다.

## 컨텍스트 전달

에이전트 간 전달에는 전체 대화 대신 다음의 최소 패킷을 사용한다.

```text
목표:
범위와 제외 범위:
관련 요구사항 ID:
관련 파일:
제약과 결정:
검증 명령:
기대 산출물:
```

이를 통해 도구마다 다른 에이전트 기능을 사용해도 동일한 결과 계약을 유지한다.

## 독립 검증 원칙

- 리뷰 역할은 구현자의 설명보다 diff, 테스트 결과, 요구사항을 우선 확인한다.
- 자동 테스트 통과는 리뷰의 대체물이 아니다.
- 보안, 데이터 마이그레이션, 인증, 결제 변경은 별도 체크리스트를 적용한다.
- 에이전트 분리가 불가능한 도구에서는 구현을 완료한 뒤 컨텍스트를 분리한 검증 단계를 수행한다.

## 다중 AI adapter parity

Codex·Claude Code·GitHub Copilot adapter는 모두 이 문서의 역할 계약과
`docs/persona-and-role-guidelines.md`를 참조하며, 네이티브 agent 기능 유무와 관계없이
`sequential-baseline`을 제공한다. subagent·병렬 실행은 역할 계약의 권한을 확대하지 않으며,
지원하지 않는 도구에서는 같은 검토 단계를 순차 수행한다.

`.ai/manifests/adapter-parity.json`과 `scripts/validate-adapter-parity.mjs`는 공통 정책·역할·권한
reference, 전역 persona 부재, Codex·Claude Code hook outcome, GitHub Copilot 공통 fallback과
materialized downstream 계약을 검사한다.

이 PASS는 실제 Codex·Claude·Copilot model outcome이 동일하다는 뜻이 아니다. model·surface별
비결정 task 비교는 exact version과 비용·network 승인 뒤 별도 수행한다.
