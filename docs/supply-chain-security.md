# 스킬·플러그인 공급망 보안

상태: 제안

## 보안 목표

이 프로젝트에서 “설치 가능”은 유명하거나 유용하다는 뜻이 아니다. 승인된 정확한 버전이
텔레메트리 없이, 선언된 최소 권한으로 동작하고, 변경 내역을 추적할 수 있다는 뜻이다.

절대적인 안전을 사전 정적 검사만으로 보증할 수는 없다. 따라서 소스 검토, 버전 고정,
격리 실행, 네트워크 차단과 업데이트 재심사를 함께 사용한다.

AI 도구의 런타임 권한, prompt injection과 파괴적 작업은
[AI 보안 가드레일](ai-security-guardrails.md)에서 다룬다.

## 신뢰하지 않는 입력

다음은 모두 설치 전까지 신뢰하지 않는다.

- `SKILL.md` 안의 명령과 외부 URL
- 플러그인 훅과 MCP 서버
- `package.json` 설치 스크립트와 전이 의존성
- 마켓플레이스 매니페스트와 자동 업데이트
- Git 저장소의 브랜치 HEAD와 릴리스 자산
- `npx`, `uvx`, `pipx`처럼 다운로드와 실행을 결합한 명령

Markdown 스킬도 에이전트에게 파일 삭제, 비밀 읽기 또는 외부 전송을 지시할 수 있으므로
코드가 없다는 이유만으로 자동 승인하지 않는다.

## 설치 승인 게이트

모든 항목은 다음 순서로 심사한다.

### 1. 식별

- 공식 저장소 URL, 소유자, 라이선스를 확인한다.
- 태그 또는 Git 커밋 SHA를 정한다. 이동하는 브랜치나 `latest`는 금지한다.
- 저장소 소유권 이전, 이름 변경과 리다이렉트를 기록한다.

### 2. 정적 검사

- 설치 및 제거 스크립트, 훅, 실행 파일, MCP 설정을 전부 식별한다.
- 외부 URL, DNS, 소켓, HTTP 클라이언트, 분석 SDK와 추적 픽셀을 검색한다.
- 셸 실행, 동적 코드 평가, 권한 상승, 홈 디렉터리 쓰기와 비밀 환경변수 접근을 검색한다.
- 의존성 트리, 잠금 파일, 네이티브 바이너리와 난독화 코드를 확인한다.
- 스킬 지침 안의 파괴적 명령, 데이터 반출과 프롬프트 인젝션 위험을 검토한다.

### 3. 설치 검사

- 임시 격리 디렉터리에 설치한다.
- 가능한 경우 설치 중 네트워크를 차단하고, 필요한 다운로드 대상만 별도로 허용한다.
- 설치 전후 파일 목록을 비교해 선언하지 않은 변경을 찾는다.
- 설치 산출물의 해시를 계산해 잠금 파일에 기록한다.

### 4. 런타임 검사

- 기본 네트워크 차단 상태에서 대표 작업을 실행한다.
- 프로세스, 파일 쓰기와 외부 연결을 관찰한다.
- 텔레메트리 옵트아웃이 필요한 후보는 환경변수가 없을 때 검사를 실패시킨다.
- 민감하지 않은 테스트 저장소만 사용하고 실제 비밀정보는 제공하지 않는다.

### 5. 승인

| 결과 | 의미 |
|---|---|
| 승인 | 고정 버전을 기본 네트워크 차단 상태에서 사용할 수 있음 |
| 조건부 승인 | 특정 기능 제외, 옵트아웃 강제 또는 추가 샌드박스가 필요함 |
| 보류 | 정보나 검증이 부족함 |
| 거부 | 무단 텔레메트리, 과도한 권한, 난독화 등 정책 위반이 있음 |

승인 기록에는 검토 날짜, 검토자, 소스 커밋, 해시, 허용 권한, 네트워크 대상,
제외한 기능과 재검토 조건을 포함한다.

## 텔레메트리 정책

- 사용량, 버전, 호스트 식별자, 프로젝트 경로, 프롬프트, 코드 또는 오류 보고의 자동 전송을 금지한다.
- 원격 이미지, 폰트, 배지와 추적 가능한 자산 요청도 텔레메트리 가능성이 있는 외부 통신으로 취급한다.
- 옵트인은 사용자가 목적과 전송 데이터를 확인하고 별도로 승인한 경우에만 가능하다.
- `DO_NOT_TRACK=1` 같은 공통 변수만 믿지 않고 해당 버전의 코드가 실제로 이를 처리하는지 확인한다.
- AI 도구 본체의 서비스 통신은 플러그인 텔레메트리와 별도 항목으로 문서화한다.

## 배포 규칙

- 팀 필수 스킬은 사용자 전역 경로가 아니라 프로젝트 범위에 설치한다.
- 승인 목록 방식으로 운영하며 마켓플레이스 전체를 허용하지 않는다.
- 가능하면 심사한 Markdown과 작은 스크립트만 내부 배포 산출물로 만든다.
- 원본 라이선스, 저작권 고지와 출처를 보존한다.
- 잠금 파일 변경은 코드 리뷰와 보안 게이트를 반드시 거친다.
- 자동 업데이트는 끄고 정기적인 명시적 업데이트 PR로 관리한다.

## 업데이트와 사고 대응

- 새 버전은 신규 설치와 동일하게 재심사한다. 기존 승인은 상속되지 않는다.
- 유지관리자 또는 저장소 소유권 변경 시 즉시 재심사한다.
- 예상하지 못한 외부 통신이나 파일 접근을 발견하면 사용을 중단하고 버전을 격리한다.
- 영향받은 비밀이 있을 가능성이 있으면 폐기·교체하고, 조사 결과와 후속 방지를 기록한다.
- 승인 철회 목록을 관리해 이미 설치된 환경에서도 탐지할 수 있게 한다.

## 보안 검사기 심사 결과

### Gitleaks 8.30.1 — 승인

- 공식 release asset과 SHA-256을 macOS ARM64·Linux x64에 고정했다.
- runtime telemetry를 발견하지 못했고 local repository scan만 허용한다.

### Semgrep Community 1.168.0 — 거부

metrics를 비활성화했는데도 version 실행 중 OpenTelemetry·X509 초기화에서 crash했다. runtime
안전성과 재현성 gate를 통과하지 못해 승인 목록과 실행 경로에서 제거했다.

### Opengrep 1.22.0 — 조건부 승인

- tag `f458d7f0d52cc58eae1ca3cf3d5caf101e637519`, LGPL-2.1을 확인했다.
- GitHub의 서명된 release와 공식 asset digest를 사용한다.
- source에는 기본 version check와 legacy metrics 경로가 남아 있어 무조건 승인하지 않는다.
- `OPENGREP_ENABLE_VERSION_CHECK=0`, `SEMGREP_SEND_METRICS=off`, local rules only를 강제한다.
- `auto`, URL rules, remote output, autofix와 untrusted validator를 사용하지 않는다.
- HOME·cache를 프로젝트 `.tools/`로 격리해 사용자 전역 파일을 만들지 않는다.
- macOS ARM64 checksum은
  `6105ab78eca041fcb9f83055939088ba8f96a6cb66683ceced39652938b217ce`, Linux x64는
  `45bcd58440e397ed52c50e953ccf5948909ea77087c9186fc7d277216f62e319`이다.
- 네트워크 없는 sandbox에서 version, local rule scan, positive·negative fixture가 통과했다.

공식 출처: [Opengrep repository](https://github.com/opengrep/opengrep),
[v1.22.0 release](https://github.com/opengrep/opengrep/releases/tag/v1.22.0).

## 자동 검증에 포함할 항목

향후 `scripts/validate`는 최소한 다음을 검사한다.

- 잠금 파일에 없는 스킬·플러그인
- 승인된 커밋 및 산출물 해시 불일치
- 금지된 설치 스크립트와 알려진 텔레메트리 문자열
- 필수 옵트아웃 환경변수 누락
- 선언되지 않은 훅, MCP 서버와 외부 URL
- 사용자 전역 설치에만 존재하는 팀 필수 스킬

MCP server는 일반 package 심사에 더해 tool description poisoning, runtime tool 변경, token
passthrough, filesystem·credential scope와 network egress를 검사한다. 승인·활성화·차단 절차는
[MCP 보안과 승인 절차](mcp-security.md)를 따른다.
