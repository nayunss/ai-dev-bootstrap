# Requirements

1. Read the project requirements, engineering rules, security rules, and current handoff.
2. Separate confirmed facts, project-specific inputs, assumptions, and blocked external decisions.
3. Give every durable requirement a stable ID, observable acceptance criteria, evidence, and owner when applicable.
4. Keep implementation tasks separate from actual downstream or human validation tasks.
5. Validate requirement traceability and update the handoff in the same change.

Do not read `.env*`, enable network access, install dependencies, or approve Production.
